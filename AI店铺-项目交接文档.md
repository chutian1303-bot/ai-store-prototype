## 一、项目背景

**AI 店铺** 是品牌面向消费者的 Agent 产品形态——把传统电商店铺(tab、筛选、楼层、货架等)重构成一个以意图为中心的对话式入口。

之前的工作已经做到:
- 静态 HTML 设计稿(8 屏,两个品牌:JNBY + FUJIFILM)
- 第二轮产品框架更新(底部常驻输入框 + 取消底部 tab)
- 多次设计细节迭代(店招、anna 形象、welcome 文案、chips 顺序、滚动收起等)

**这次的目标**:从静态 demo 升级为**可交互原型**,要能真的点、真的输入、真的有意图识别和动态承接,而不是预设脚本。

---

## 二、核心理念

店铺的核心能力是**意图的表达和承接**,产品价值围绕两端展开:

### 输入侧:让意图表达更自然
- **旧体验**:用户点击 tab、拉筛选、滑楼层,自己找
- **新体验**:
  - 自然语言输入(底部输入框,任何时刻可问)
  - 上下文意图预测(从内容场景进店,如"JNBY 春季上新"帖子,系统直接识别"想看上新")

### 输出侧:让意图承接更直接
- **旧体验**:漏斗式跳转,各种页面之间跳来跳去,每次跳都有折损
- **新体验**:多模态 canvas 承接,同一页面根据意图类型动态切换内容形态,不跳页面

### 四种意图类型 × 承接形态
| 意图类型 | 典型场景 | 承接形态 |
|---|---|---|
| 精准意图 | "M 码的白色风衣" | 商品货架 |
| 模糊意图 | "适合约会的" | Look 搭配 + 场景图 |
| 决策型意图 | "适合一个人旅行的相机" | 客观评测信息 |
| 服务型意图 | "怎么退换货" | 对话式导购 |

## 三、产品框架(当前版本)

输入框统一放在页面底部、底部 tab 全部去掉。用户进店后的主要环节:

### 1. Welcome 阶段
- anna(专属导购)作为消息发送主体,发欢迎词 + 店铺动态
- 根据进店上下文做意图预测,展示对应承接(例:从上新内容进店 → 直接展示新品 feeds)
- **关键细节**:anna 不放在卡片里,而是作为消息主体,头像 + 名字 + 消息分开布局
- 文案示例:"欢迎进店,最近上新了 **28 款新品**,我是你的专属店铺导购 anna,结合你平时的穿衣风格,已经为你整理了新品推荐,你可以随时找我 👋"
- "28 款新品" 用淘宝橙强调色

### 2. 沉浸浏览阶段
- 双列商品 feeds(沿用现有 HTML 设计)
- 滚动时:顶部店招 + 底部输入框同步用 `max-height` 折叠真实塌陷,不留白;停止滑动 0.9 秒后两者同步恢复

### 3. 输入框提问
- 底部常驻输入框
- 上方推荐意图 chips,顺序:`👣 商品足迹(首位)` → `找优惠券` → `穿搭推荐` → `AI 试衣`
- 用户输入后,chips 联动高亮,feeds 略暗退后,anna 在 canvas 顶部以消息形式回复(文字 + 商品 feeds)

### 4. 商品详情浮层
- 不跳页面,商品详情作为浮层在 feeds 之上打开
- 接近全屏(顶部留一点 feeds 露出)
- 展示:商品图、用户相关信息(已购系列、风格匹配等)、购买按钮
- 底部输入框依然常驻,推荐意图 chips 切换为商品相关:`尺码推荐` / `AI 试衣` / `搭配建议` 等

### 5. 关闭浮层后
- 输入框上方最左侧出现 **【店内足迹】** 入口,点进去是看过的商品列表

## 四、关键设计决策(已定,无需再讨论)

### 店招规范
- LOGO:JNBY
- 行 1:`JNBY 江南布衣旗舰店 · ⭐4.8 · 62.3 万粉丝`
- 行 2:标签 `官方旗舰店` `品牌设计师` `白领精选`
- **不要** 在店招里写 "anna 专属客服"(welcome 已说过)

### anna 形象
- 名字:anna(小写)
- 定位:你的专属导购
- 作为消息发送主体出现,有头像 + 名字
- 不要包在 "卡片" 里,要像聊天消息

### 颜色
- 强调色:淘宝橙(不要用黑底高亮)
- 整体偏 JNBY 的设计师品牌调性,留白多

### Chips 顺序(已定)
足迹 → 找优惠券 → 穿搭推荐 → AI 试衣

## 五、两个品牌场景(用于验证跨品类通用性)

### JNBY 江南布衣(时装)
- 意图特点:**感性、模糊**("适合约会的"、"通勤穿什么")
- 承接重点:Look 搭配、场景图、AI 试衣
- 商品 feeds 示例:LESS 廓形短外套、LESS 春季薄风衣、亚麻解构西装外套等

### FUJIFILM 富士(相机)
- 意图特点:**理性、决策型**("适合一个人旅行的相机"、"vlog 拍人哪款好")
- 承接重点:客观评测信息、参数对比、使用场景推荐
- 验证目标:同一套意图框架能否覆盖理性决策类品类

## 七、待决事项(下一步要和 Cowork 一起讨论)

1. **可交互的"程度"**
   - 选项 A:点得动的高保真原型,所有 AI 回答用预设脚本(state 机)
   - 选项 B:接 Anthropic API,真实做意图识别和承接生成
   - 推荐 B,因为产品的灵魂就是真实意图识别

2. **技术栈**
   - 推荐 Vite + React + TypeScript + Tailwind
   - 移动端优先,手机框模拟容器

3. **第一阶段范围**
   - 建议先做 JNBY 完整链路(5 个环节),跑通再复用到 FUJIFILM
   - 第一屏目标:Welcome → 输入"给我推荐同款风衣" → anna 回复 feeds → 点商品出浮层 → 关闭浮层看足迹

4. **数据来源**
   - 商品数据先用 mock(JSON 静态文件)
   - 后续可考虑接 mock server 或真实电商 API

5. **意图识别实现**
   - 如选 B,需要设计 prompt:输入用户原始 query,输出 `{intent_type, parsed_filters, suggested_response_format}`
   - 这部分让 Cowork 第一阶段就实现一个最小可用版本

## 八、产品 owner 的一些偏好(从过往迭代里看出来的)

- 喜欢 **真实塌陷** 而不是仅仅隐藏(滚动收起那种细节会反复要求)
- 强调 **消息感** 而不是 **卡片感**(welcome 要拆开,不要包卡片)
- 强调色用品牌色或电商常规色(淘宝橙),不喜欢黑底
- 顶部信息要紧凑,不要重复(anna 已经说过的就不在店招里再说)
- 喜欢用真实数据填充(粉丝数 62.3 万这种)
